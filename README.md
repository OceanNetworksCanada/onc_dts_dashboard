# ONC DTS Dashboard

Realtime dashboard for Distributed Temperature Sensing (DTS) data from a Silixa XT DTS
instrument, streamed directly from Ocean Networks Canada's Oceans 3.0 API.

**Stack:** Vue 3 + Vite → GitHub Pages. No backend, no database, no build secrets — the
browser fetches and decodes DTS data directly from ONC's API using a token you provide.

## How it works

DTS measures temperature along a fiber-optic cable: each measurement is a profile of
temperature vs. distance-along-fiber, alternating between two channels roughly every 10
seconds. This dashboard shows:

1. **Latest profile** — temperature vs. distance, live, per channel.
2. **Waterfall heatmap** — time × distance × temperature.
3. **Point time series** — temperature over time at fiber positions you pick.
4. **Map** — the cable route colored by current temperature.

## Bring your own token

Each visitor pastes their own [ONC API token](https://data.oceannetworks.ca/Profile) (under
"Web Services API"). The token is stored only in your browser's `localStorage` and sent only
to `data.oceannetworks.ca` — it is never sent anywhere else, and nothing is stored server-side
because there is no server.

## Quickstart

```bash
npm install
npm run dev
```

Open the printed local URL, paste your ONC API token, and connect. If the instrument is
currently offline, use the "Start from" field to backfill from a known historical timestamp
instead of the live edge.

## Build / deploy

```bash
npm run build      # outputs to dist/
npm run preview    # serve the production build locally
```

Pushing to `main` runs `.github/workflows/deploy.yml`, which builds and publishes `dist/` to
GitHub Pages. No repository secrets are required.

## Cable geometry

`src/assets/cable-route.geojson` is generated by `scripts/generate-cable-route.mjs` from an
origin point + azimuth per channel (a placeholder projection, not a survey). Regenerate it as
better geometry becomes available:

```bash
node scripts/generate-cable-route.mjs
```
